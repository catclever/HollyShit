# Phase 1 完整落地设计：多视角内生空间（Multi-View Endogenous Space）

> 该设计文档详细阐述了如何从单一 Mock 空间跨越到基于真实多维特征矩阵的流形对齐。

## 1. 核心挑战与架构野心 (The Challenge & Goal)
在 Phase 1 最初的 POC 验证中，系统以硬编码的微型网络成功证明了“概率网覆盖+动量牵引+愚蠢解码器”的闭环是通顺的。
进入完整体落地后，我们拿到了 4 份极其珍贵的真实预训练空间特征（RoBERTa:768维, Text2Vec:768维, GTE:1024维, BGE:1024维）。
我们的架构野心（Goal）绝*不仅仅*是做一个“支持多种输入的万能转换头”，而是**要利用这 4 位“见识过大千世界的上帝导师”，通过“神罚重构”，蒸馏并锻造出一个具有绝对主权的、不依赖任何一家的“Mamba 最终原生空间 (z-space)”**。

## 2. 翻译官矩阵：Multi-Emb Adapter
面对维度不一的数据（768 与 1024），在数学上不能直接求均值融合，必须搭建桥梁。

### 2.1 维度统一器 (Dimension Unifiers)
我们不再使用 `[batch, seq, dim]` 的单管线输入，而是构建一个包含 **4 个并行 `GodEncoder`（各自为 3 层 SiLU 激活的 MLP）** 的大型接线板 `MultiEmbAdapter`。
不论外部输入的维度是 768 还是 1024，都会被各自专属的非线性网络“强行扭曲并压缩（或扩展）”到 Mamba 绝对领地的统一坐标：**`z_dim = 1024`**。

### 2.2 拓扑展平器与 GodEncoder 详细设计 (Topology Flatteners)
`GodEncoder` 的输入是一个序列张量 `[Batch, Seq_len, raw_dim]`。
它的核心结构是**非线性多层感知机（MLP，例如 3 层的 Linear 配合 SiLU 激活函数）**，对每一个 Token 进行逐帧的几何变换：
1. **解构（Linear 1）**：把外部异构空间（比如对比学习特有的各向异性、聚类尖刺等）打碎。
2. **重塑（SiLU 等）**：引入非线性，使得原本外部空间的线性关系被扭转成 Mamba 需要的新物理关系。
3. **入轨（Linear 3）**：最后稳稳地把特征锚定到 Mamba 的绝对坐标系 `z_dim = 1024` 中。
每个 `GodEncoder` 就像一个极其强大的几何曲面贴合器，消除不同外部模型的固有偏见，把它们拉平为适合 WeakDecoder 阅读的“光滑纯净流形”。

## 3. 多视角融合策略：核心权重轮换融合 (Core-Weighted Fusion)
经过统一压缩后，我们得到了 4 个齐整的、高维同构张量 $z_1, z_2, z_3, z_4$。下一步是决定给 Mamba 吃什么。
基于您的绝佳建议，**我们抛弃“全有或全无（All-or-Nothing）”的激进做法，而是采用“核心决定 + 侧翼辅助”的动态权重轮换法**。

#### 数学机制：
确实是**按 Batch 为颗粒度**进行的：在给定的一个 Batch 里，系统随机抛骰子，抽中一种 Emb（假设抽中索引 2, 即 GTE）作为该回合的**决定性核心（Decisive Core）**，赋予压倒性的权重（例如 $\alpha = 0.7$）。
剩下的 3 种 Emb 退居二线成为附属特征，平分剩余的少数权重（例如分别占据 $0.1$）：
$$ \mu_{target} = 0.7 \cdot z_{gte} + 0.1 \cdot z_{bge} + 0.1 \cdot z_{roberta} + 0.1 \cdot z_{text2vec} $$

#### 深远物理意义：
1. **提供引力底座，防止空间断崖（Stabilized Manifold）**：如果我们做极端的全丢弃，上个 Batch 和这个 Batch 的宇宙法则会发生剧烈的跳跃，Mamba 容易精神分裂。保留 30% 的全局辅助权重，相当于给任何一次融合都提供了一个极其稳固的“全视角锚点底座”，让 Loss 曲线平滑跌落。
2. **主次分明，打破协作性重度依赖（Curing Co-adaptation）**：因为 70% 的信息量都在当选的那位核心 Emb 身上，该回合被选为主演的 `GodEncoder` 仍旧被迫承担起 100% 的独立翻译责任。它不能指望那 3 位各占 0.1 戏份的配角帮忙，必须单枪匹马学出完美的映射。同时，配角们也能分到少量的“微光梯度滋养”，保证 4 个 Encoder 的坐标体系长在同步进化的节奏上。

## 4. 序列幽灵的驱散：Masked Loss 升级
进入真实语料意味着所有的张量带有 Padding 字符。若再无脑对全空间求 `.mean()`，会导致整个物理空间向 Zero 坍缩。必须从数学层面修改：

1. **绝对主张覆盖率 $\mathcal{L}_{coverage}$**：必须乘入 Attention Mask。我们不需要在一堆毫无意义的 $<PAD>$ 空白处去撒概率网。
2. **动量连贯性惩罚 $\mathcal{L}_{momentum}$**：动量是在两个相邻时间序列坐标之间求 Delta。连续的 Padding Token 意味着静止不动，会产生零向量。这在计算余弦相似度时会导致除零 NaN 爆炸，甚至形成不可逃逸的黑洞动量陷阱。计算步长必须在有效文本截断点之前终止。
3. **愚蠢译码器 $\mathcal{L}_{recon}$**：Label CrossEntropy 必须施加 Mask 忽略占位符。

## 5. 远景演化阶段：卸磨杀驴 (Future Endpoint)
这份设计的终极畅想恰恰在于**“当大厦落成，便拆除外部脚手架”**。

这 4 个不同维度的庞大数据源，以及专门打点适配的 `MultiEmbAdapter`，在这个极其优雅的非线性动力网络成型后，其全部知识已经被死死刻入了 Mamba 的参数空间中。
- 到那时，外部的 4 位神明统统下岗。
- 我们只需要接入一个极其迷你的、原生随机初始化的 `Native Embedding` 层（即 Token ID $\rightarrow z_0$ 字典矩阵）。
- 系统进入自给自足的闭环时代：外界给出一句话，我们从原生词典拿到 $z_0$，Mamba 自己往前滑翔，一切不再借助半分外力。

这就是 Phase 1 完整体在架构和物理层面实现“空间脱敏”的全景体系。
